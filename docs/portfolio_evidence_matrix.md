## Deep Review Use

If challenged on depth, answer in this order:

1. State the invariant.
2. Name the application code that enforces it.
3. Name the eval or smoke flow that proves it.
4. State what would change in production without changing the invariant.

Example:

```text
For permission-aware RAG, the invariant is that unauthorized evidence never reaches the model. Retrieval filters documents by user role before answer generation, the Alice finance eval proves abstention, and the Morgan finance eval proves authorized access still works. In production, JSON storage would be replaced with PostgreSQL row-level security and vector retrieval, but the permission boundary would stay before model generation.
```
