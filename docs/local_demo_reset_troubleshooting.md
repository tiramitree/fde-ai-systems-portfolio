# Local Demo Reset Troubleshooting

Use this guide when a local demo looks stale after repeated runs, evals, browser edits, or replay checks. The source of truth stays in checked-in fictional seed files and `docs/demo_state_presets.json`; generated runtime state and browser drafts are local evidence only.

## Fast Reset

From the repository root, start every demo with reset state:

```bash
python -B scripts/dev.py start
```

That command launches all three services with `app.py --reset`:

```bash
cd secure-enterprise-knowledge-copilot
python -B app.py --reset --port 8765
```

```bash
cd regulated-customer-operations-agent
python -B app.py --reset --port 8770
```

```bash
cd ai-reliability-incident-console
python -B app.py --reset --port 8780
```

If a port is already in use, stop the old local demo process before restarting. Do not delete unrelated Python processes.

## What Gets Reset

| Surface | Local Artifact | Reset Behavior |
| --- | --- | --- |
| Demo runtime state | `*/data/runtime_state.json`, `*/data/runtime_state.tmp` | Rebuilt from checked-in fictional seed data when the service starts with `--reset`. |
| Eval runtime state | `*/data/eval_runtime_state.json` | Rebuilt by eval runners from checked-in eval fixtures; it is evidence, not source truth. |
| Replay evidence | `out/demo_replay_artifact.*`, `docs/demo_report.md`, `otel_traces.json`, `eval_summaries.csv` | Regenerated by their commands; review diffs before committing intentional release evidence. |
| Browser scenario drafts | `localStorage` keys beginning with `fde-scenario-draft:` | Browser-local only. Clear from the scenario draft panel or browser developer tools when a saved draft masks the seed snapshot. |

The three scenario draft keys are:

- `fde-scenario-draft:secure-enterprise-knowledge-copilot`
- `fde-scenario-draft:regulated-customer-operations-agent`
- `fde-scenario-draft:ai-reliability-incident-console`

Browser theme preferences also use `localStorage`, but they do not affect seed data, eval data, traces, approvals, or release decisions.

## Canonical Reset Presets

`docs/demo_state_presets.json` is the shareable reset map for the three canonical local paths:

- `p1-finance-access`: Alice must abstain from the finance document, while Morgan receives the cited finance answer.
- `p2-case-1001-approval`: Ivy creates a governed approval request for `case-1001`, bypass is refused, and Sam approves the side effect.
- `p3-unsafe-canary-release`: Maya triages the unsafe canary release and rollout is blocked by failed eval evidence.

Verify the presets whenever reset behavior, seed fixtures, eval fixtures, API payloads, or demo docs change:

```bash
python -B scripts/dev.py demo-presets
```

## Symptom Map

| Symptom | Likely Cause | First Fix |
| --- | --- | --- |
| A changed approval id, trace id, or audit event appears after a previous run. | The running service is using existing `runtime_state.json`. | Stop the service and restart with `python -B scripts/dev.py start`. |
| A scenario draft panel shows edited JSON instead of the checked-in seed snapshot. | Browser `localStorage` restored a saved `fde-scenario-draft:*` value. | Use the panel's clear action, or remove the matching key in browser developer tools. |
| Eval summary rows do not match expected seed behavior. | `eval_runtime_state.json` or generated summaries are stale. | Run `python -B scripts/dev.py evals`, then `python -B scripts/dev.py claims` if public metrics changed. |
| Fresh-clone verification passes locally but the browser still looks different. | Browser state survived outside the repository checkout. | Clear the matching scenario draft `localStorage` key and refresh the page. |

## Files Not To Commit As Source

Do not add generated local state as source content:

- `*/data/runtime_state.json`
- `*/data/eval_runtime_state.json`
- `*/data/runtime_state.tmp`
- `out/demo_replay_artifact.*`
- `otel_traces.json`
- `eval_summaries.csv`
- local browser `localStorage` exports

Generated evidence should be committed only when a release process explicitly asks for it and the diff is reviewed. Do not add secrets, private paths, real customer data, external accounts, paid-service requirements, generated runtime files, or real incident details.

## Verification Path

After a reset-facing change, use the narrow gate first:

```bash
python -B scripts/dev.py demo-presets
```

Then run the public-safety and release-facing checks:

```bash
python -B scripts/dev.py safety
python -B scripts/dev.py quality
python -B scripts/dev.py fresh-clone-local
```

For artifact naming and command output triage, keep this guide aligned with `docs/local_artifact_glossary.md` and `docs/command_output_troubleshooting_map.md`.
